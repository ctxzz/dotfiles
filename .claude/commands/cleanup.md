# コードベースクリーンアップ

## ファイル整理
1. **未使用ファイル**
   - 孤立したファイルの特定
   - 古いテストフィクスチャの削除
   - 一時ファイルのクリーンアップ
   - 非推奨モジュールのアーカイブ

2. **コード品質**
   - コメントアウトされたコードの削除
   - TODOアイテムの修正
   - 古いコメントの更新
   - 変数名の改善

## dotfiles特有のクリーンアップ
1. **設定ファイル**
   - 重複する設定の統合
   - 未使用の設定項目の削除
   - バックアップファイルの整理
   - .oldや.backupファイルの確認

2. **シンボリックリンク**
   - 壊れたシンボリックリンクの検出
   - 不要なシンボリックリンクの削除

3. **スクリプト**
   - 未使用の初期化スクリプトの削除
   - 古いインストールスクリプトの整理
   - デバッグ用のecho文の削除

## Git クリーンアップ
- 追跡されていないファイルの削除: !`git clean -fd --dry-run`
- 削除されたブランチのプルーン: !`git remote prune origin`
- .gitignoreの更新確認

## 注意事項
- 削除前にバックアップの確認
- 重要なファイルは削除しない
- git clean実行前にdry-runで確認
